**Step 2: Create normalize_answer module.**

Create `workspace/src/tools/normalize_answer.py` with a single public function:

```python
def normalize_answer(raw: str) -> dict:
```

**Input validation:** If `raw` is None or not a string, return `{"error": "INVALID_INPUT", "reason": "raw must be a non-empty string"}`.

**Decision tree (from RESEARCH.md):**
1. Strip leading/trailing whitespace.
2. Normalize unicode minus (`\u2212`) to ASCII `-`.
3. If starts with `[` -> pass-through (list answer). Return `{"result": stripped}`.
4. If contains unit word (`million`, `billion`, `thousand` -- case insensitive, with or without trailing 's') -> pass-through. Return `{"result": stripped}`.
5. If starts with `$` -> pass-through. Return `{"result": stripped}`.
6. If matches date pattern (month name followed by digits, or month name + year) -> pass-through. Return `{"result": stripped}`.
7. If ends with `%` -> percentage: strip `%`, preserve the number exactly as-is (including decimal places), re-append `%`. Return `{"result": formatted}`.
8. If contains `.` (decimal) -> preserve exact decimal places from raw string. Do NOT strip trailing zeros. Return `{"result": stripped}`.
9. Else -> integer: preserve commas if present in raw, do not add commas if absent. Return `{"result": stripped}`.

**Key implementation notes:**
- The normalizer is fundamentally a pass-through for format -- its job is to clean whitespace and unicode, not to reformat numbers. The benchmark answers ARE the expected format.
- For categories A-F (numeric), the normalizer ensures the raw value is clean (whitespace, unicode) but preserves the exact numeric format.
- Never call `Decimal.normalize()` -- it strips trailing zeros.
- The `result` field is always a string.

Return format: `{"result": str}` on success, `{"error": str, "reason": str}` on failure.

```python
import json
import pytest
from pathlib import Path

FIXTURE = Path(__file__).parent / "fixtures" / "format_survey.json"

def load_survey():
    with open(FIXTURE) as f:
        data = json.load(f)
    cases = []
    for cat_name, cat in data["categories"].items():
        for ex in cat["examples"]:
            cases.append((ex["raw"], ex["expected"], cat_name))
    for ec in data["edge_cases"]:
        cases.append((ec["raw"], ec["expected"], ec.get("note", "edge")))
    return cases

@pytest.mark.parametrize("raw,expected,category", load_survey())
def test_normalize_matches_benchmark(raw, expected, category):
    from src.tools.normalize_answer import normalize_answer
    result = normalize_answer(raw)
    assert "result" in result, f"Expected success for {category}: {raw}, got {result}"
    assert result["result"] == expected, f"Category {category}: normalize({raw!r}) = {result['result']!r}, expected {expected!r}"
```

**Additional explicit tests:**
- `normalize_answer(None)` -> INVALID_INPUT error
- `normalize_answer("")` -> INVALID_INPUT error
- `normalize_answer(123)` -> INVALID_INPUT error (not a string)
- Unicode minus: `normalize_answer("\u2212507")` -> `{"result": "-507"}`
- Trailing zeros: `normalize_answer("11.60")` -> `{"result": "11.60"}` (not "11.6")
- Large integer no comma: `normalize_answer("935851121560")` -> `{"result": "935851121560"}`
- Integer with comma: `normalize_answer("2,602")` -> `{"result": "2,602"}`
- Percentage 3dp: `normalize_answer("13.009%")` -> `{"result": "13.009%"}`
- Dollar pass-through: `normalize_answer("$37,921,314")` -> `{"result": "$37,921,314"}`
- List pass-through: `normalize_answer("[0.096, -184.143]")` -> `{"result": "[0.096, -184.143]"}`
- Unit word: `normalize_answer("36080 million")` -> `{"result": "36080 million"}`
- Date: `normalize_answer("March 3, 1977")` -> `{"result": "March 3, 1977"}`

The parametrized test ensures every single example from the survey fixture is verified. This is the CAL-06 requirement: "test suite asserts normalize(raw) == expected for all distinct answer format patterns observed in the survey."
