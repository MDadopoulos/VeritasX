```python
def extract_table_block(span_text: str, anchor: str) -> dict:
```

**Input validation:** If `span_text` is empty/whitespace or `anchor` is empty/whitespace, return `{"error": "INVALID_INPUT", "reason": "..."}`.

**Algorithm (from RESEARCH.md Pattern 4):**
1. Split span_text into lines.
2. Case-insensitive search for `anchor` in each line.
3. From the anchor line, look ahead up to 15 lines for the first line starting with `|` (the table start).
4. If no `|` line found within 15 lines of any anchor match, return `{"error": "NO_TABLE_FOUND", "reason": "No pipe-delimited table found within 15 lines of anchor"}`.
5. **Backward walk** from table start (up to 4 lines): look for unit annotation matching `^\s*\((?:cumulative\s*[-–]\s*)?in\s+\w[\w\s]*of\s+\w[\w\s]*\)` (case-insensitive). Also capture the first non-blank prose line before the unit annotation (or before the table if no unit annotation).
6. **Forward walk** to collect all consecutive `|`-prefixed lines (the table body).
7. **Footnote walk** after table body: collect lines that are footnotes (`\d+[/]` prefix, `Source:` prefix, or continuation lines). Stop at blank line followed by non-footnote line.
8. After collecting the first table block, continue scanning the remaining lines for additional `|` tables (per CONTEXT decision: return all candidate tables as a list).
9. Return: `{"tables": [{"prose_context": str, "unit_annotation": str|None, "table_text": str, "footnotes": str, "start_line": int, "end_line": int}, ...]}` -- start_line/end_line are 1-indexed.

**Important per CONTEXT decisions:**
- Always include footnote rows -- they carry unit metadata critical for arithmetic.
- Include the one prose sentence immediately preceding the table.
- Multi-row column headers: include all header rows as-is, do not merge.
- The 15-line limit is for finding the table START only, not the block size. Once found, include all rows.

Create `workspace/src/tools/classify_table_rows.py` with a single public function:

```python
def classify_table_rows(table_text: str) -> dict:
```

**Input validation:** If `table_text` is empty/whitespace, return `{"error": "INVALID_INPUT", "reason": "..."}`.

**Algorithm (from RESEARCH.md Pattern 5):**
1. Split `table_text` by newlines. Each line should start with `|`.
2. For each `|` row, split by `|` to extract cells. The first non-empty cell is the row label.
3. Classify each row into one of three buckets:
   - **header_rows**: rows matching header patterns (separator `---`, "fiscal year or month", "calendar year", all-NaN, label ending with `:`) -- OR any row before the `| --- |` separator row.
   - **aggregate_rows**: rows whose label matches aggregate patterns (bare year `^\d{4}$`, year with note `^\d{4}\s*\(`, `total`, `annual`, `fiscal year`, `cal.`, `to date`, `average`, `cumulative`, `estimate`). Apply aggregate detection FIRST (higher specificity per CONTEXT decision).
   - **month_rows**: rows whose label matches month patterns (full or abbreviated month name, with optional YYYY- prefix).
4. For data rows (month + aggregate), parse numeric cell values using `parse_cell_value()` helper (strip footnote suffixes like `3/`, `14/`, trailing `r`; handle parenthetical negatives `(1234)` -> `-1234`; strip commas; convert to `Decimal`; return `None` for null cells `-`, `nan`, `N/A`, etc.).
5. Track year inheritance: when a row has `YYYY-MonthName` pattern, capture `YYYY`; propagate to subsequent month rows that have no year prefix until a new year prefix appears.
6. Return:
```python
{
    "header_rows": [{"label": str, "raw": str}],
    "month_rows": [{"label": str, "year": str|None, "month": str, "values": [Decimal|None, ...], "raw": str}],
    "aggregate_rows": [{"label": str, "values": [Decimal|None, ...], "raw": str}]
}
```

Include `parse_cell_value` as a module-level helper (not private -- it will be used by the calculator in Phase 3 for direct cell parsing). Import `Decimal` from `decimal`, `re` from stdlib.
