  <action>
Create `workspace/src/tools/calculate.py` with three public functions. Follow the code patterns from RESEARCH.md exactly.

**Function 1: `calculate(expr: str) -> dict`**

Per CAL-01:
1. Input validation: empty/whitespace -> `{"error": "INVALID_INPUT", "reason": "Expression is empty"}`.
2. `ast.parse(expr.strip(), mode='eval')` -- catch SyntaxError -> `{"error": "SYNTAX_ERROR", "reason": str(e)}`.
3. Walk all nodes with `ast.walk(tree)`. If any `type(node) not in SAFE_NODES`, return `{"error": "DISALLOWED_NODE", "reason": f"AST node {type(node).__name__} not allowed"}`.
4. SAFE_NODES = `{ast.Expression, ast.BinOp, ast.UnaryOp, ast.Add, ast.Sub, ast.Mult, ast.Div, ast.Pow, ast.Mod, ast.UAdd, ast.USub, ast.Constant}`.
5. Recursively evaluate using `_eval_node()` with `Decimal(str(node.value))` for Constants. CRITICAL: use `str()` to avoid float contamination.
6. Division by zero -> `{"error": "DIVISION_BY_ZERO", "reason": "Denominator is zero"}`.
7. Success -> `{"result": Decimal_value}`.

Set `getcontext().prec = 28` at module level.

**Function 2: `pct_change(old, new, unit_old: str = None, unit_new: str = None) -> dict`**

Per CAL-02 + CAL-04:
1. Accept any numeric type (str, int, float, Decimal). Convert both to `Decimal(str(old))`, `Decimal(str(new))`. InvalidOperation -> `{"error": "INVALID_INPUT", "reason": "..."}`.
2. If `unit_old` and `unit_new` are both provided and non-None, check for unit mismatch. Normalize units (strip trailing 's', lowercase). If they differ (e.g., "million" vs "billion"), return `{"error": "UNIT_MISMATCH", "reason": f"Cannot compute pct_change: {unit_old} vs {unit_new}"}`. If either unit is None/empty, skip check (per CONTEXT: unlabeled values pass through).
3. If `old_d == 0`, return `{"error": "DIVISION_BY_ZERO", "reason": "old value is zero, cannot compute pct_change"}`.
4. Compute `(new_d - old_d) / old_d * Decimal('100')`.
5. Round to 2 decimal places: `round(result, 2)`.
6. Return `{"result": rounded_value}`.

**Function 3: `sum_values(pairs: list[tuple[str, any]], expected_count: int) -> dict`**

Per CAL-03 + CAL-04:
1. If `len(pairs) != expected_count`, return `{"error": "COUNT_MISMATCH", "reason": f"Expected {expected_count} pairs, got {len(pairs)}", "actual_count": len(pairs)}`.
2. Sum all values, converting each to `Decimal(str(value))`. If any conversion fails, return `{"error": "INVALID_INPUT", "reason": f"Cannot parse value for label {repr(label)}"}`.
3. Extract unit words from labels using `\b(millions?|billions?|thousands?)\b` regex. Normalize (lowercase, strip trailing 's').
4. If multiple distinct units found, include `"unit_warning": f"Heterogeneous units in labels: {sorted(units)}"` in result (per CONTEXT: warn but do not reject).
5. Return `{"result": total, "pair_count": len(pairs)}` (plus unit_warning if applicable).

All three functions follow the error contract: return dict, never raise, never return None.
