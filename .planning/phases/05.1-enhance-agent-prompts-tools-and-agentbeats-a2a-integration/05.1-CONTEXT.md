# Phase 5.1: Enhance Agent Prompts, Tools and AgentBeats A2A Integration - Context

**Gathered:** 2026-03-21
**Status:** Ready for planning

<domain>
## Phase Boundary

Improve existing agent accuracy and A2A compliance. Three tracks:
1. **Prompt improvements** — rewrite system prompt to encode corpus-specific retrieval knowledge
2. **Tool description improvements** — add corpus structure hints to tool docstrings; enable parallel tool calls
3. **AgentBeats A2A integration** — align server architecture with the RDI Foundation agent-template and AgentBeats purple-agent spec, including Docker

No new tool capabilities. No new pipeline stages. Pure quality and integration work.

</domain>

<decisions>
## Implementation Decisions

### Prompt strategy
- Root failure mode is **bad retrieval choices** — agent doesn't look in adjacent-year bulletins
- Core rule to hardcode in system prompt: FY data often appears in the *following* calendar year bulletin (e.g. FY1940 data is in the 1940–1941 bulletin). Always check the file for year N AND year N+1 when searching for FY N data.
- Researcher should look up best practices for RAG retrieval prompting with domain-specific corpora (online research)
- "Cannot determine" give-up policy: already handled by verifier — do NOT duplicate in system prompt

### Tool improvements
- No new tools needed — existing toolset is sufficient
- Improvements are in **tool descriptions** (docstrings the LLM sees), not tool logic
- Each tool's description should include **corpus structure hints** — e.g. route_files explains fiscal-year-to-bulletin mapping
- Parallel tool call strategy: Claude's discretion — determine what makes sense to parallelize (likely: route_files for multiple year candidates, or search_in_file across multiple files simultaneously)

### A2A integration
- Target architecture: RDI Foundation agent-template (https://github.com/RDI-Foundation/agent-template) — Docker + file server + executor + agent messenger + REST
- Follow AgentBeats purple-agent tutorial: https://docs.agentbeats.dev/tutorial/#2-purple-agents
- Researcher should analyze the template and map existing components vs. what needs building — Claude's discretion on rewrite vs. adapt
- Docker scope: local dev first, then push/deploy following AgentBeats guidelines (not full Cloud Run production setup)

### Claude's Discretion
- Parallel tool call strategy (which calls to parallelize)
- Whether to rewrite current FastAPI server or adapt it to match the agent-template architecture
- Exact corpus structure hint wording in tool docstrings

</decisions>
