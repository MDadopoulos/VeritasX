## What Was Built

### Task 1 — SYSTEM_PROMPT Retrieval Rules

Two new mandatory sections were inserted into `SYSTEM_PROMPT` in `workspace/src/agent.py`, positioned after "Mandatory Planning Gate" and before "Scratch File Writing Instructions":

**Fiscal Year Adjacency Rule (MANDATORY):** Instructs the agent that FY N data may appear retrospectively in year N+1 bulletins. Requires calling `route_files` twice — once for the original question (year N) and once for "fiscal year N+1" — and searching both sets before concluding data is absent.

**Parallel Search Rule:** Instructs the agent to call `search_in_file` for ALL file paths returned by `route_files` in a SINGLE turn (parallel tool calls) rather than sequentially.

No cannot-determine give-up logic was added (locked decision — verifier handles this). The existing Verification and Retry Protocol section is unchanged.

### Task 2 — Tool Docstring Corpus Hints

Each tool module's primary public function docstring was extended with corpus-specific guidance the LLM sees when the tool is bound:

- **route_files:** `CORPUS STRUCTURE` block describing bulletin naming convention, date range, FY span definition, and retrospective-publishing warning with explicit instruction to call a second time for year N+1.
- **search_in_file:** `CORPUS STRUCTURE` (table format, 20-line spans, unit annotation placement), `PARALLEL CALL GUIDANCE` (issue all calls simultaneously), and `QUERY TIPS` (include series name + year, try "fiscal year" and "annual" for FY totals).
- **calculate:** `CORPUS CONTEXT` on `calculate()`, `pct_change()`, and `sum_values()` — all prohibit inline arithmetic and direct the agent to use the tool for every computation.
- **normalize_answer:** `USAGE` note stating the tool must only be called after verifier PASS, and that `verification_token` is mandatory.
- **verifier:** `CORPUS CONTEXT` added to `resolve_era_column_header()` describing the four dimensions the verifier validates.
