## Decisions Made

- **python:3.11-slim over uv:** Project uses requirements.txt, not pyproject.toml — pip-based approach has lower friction and matches the existing dev environment.
- **Corpus as volume, not COPY:** Corpus is large and changes independently of code. The Dockerfile includes a commented-out COPY line documenting how to build a fully self-contained image, but the default design expects corpus to be mounted.
- **amber-manifest.json5 image placeholder:** The `image` field contains `ghcr.io/YOUR_ORG/officeqa-agent:latest` — this is intentional and documented. The actual registry URL will be set when the CI/CD pipeline is configured.
- **Non-root user:** Using `useradd -m agent` and `USER agent` follows the security pattern from agent-template and reduces container attack surface.
