**Create workspace/Dockerfile** using pip-based approach (lower friction than uv, per research recommendation):

```dockerfile
# OfficeQA Finance Agent — A2A-compatible Docker image
FROM python:3.11-slim-bookworm

# Create non-root user
RUN useradd -m agent
WORKDIR /home/agent/workspace

# Install dependencies first (layer caching)
COPY requirements.txt ./
RUN pip install --no-cache-dir -r requirements.txt

# Copy source code
COPY src/ src/
COPY scratch/ scratch/

# Copy corpus (large — placed last for cache efficiency during dev)
# Note: corpus is mounted as volume in local dev; COPY is for full image builds
# COPY ../corpus/ ../corpus/

# Set environment defaults
ENV SERVER_PORT=9009
ENV SERVER_HOST=0.0.0.0
ENV CORPUS_DIR=/home/agent/corpus/transformed

# Expose A2A port
EXPOSE 9009

# Run the A2A server
USER agent
ENTRYPOINT ["python", "-m", "src.server"]
```

**Key decisions:**
- Python 3.11-slim (matches dev environment, no uv complexity)
- pip-based install (project uses requirements.txt, not pyproject.toml)
- Non-root user `agent` (security best practice from agent-template)
- CORPUS_DIR defaults to /home/agent/corpus/transformed — in production the corpus is mounted as a volume; for self-contained images, uncomment the COPY line
- scratch/ directory is pre-created but will be overwritten per-question
- ENTRYPOINT uses `python -m src.server` which triggers the `if __name__ == "__main__"` block in server.py

**Verify the Dockerfile builds** (requires Docker installed):

**Create workspace/amber-manifest.json5** for AgentBeats registration:

```json5
// AgentBeats registration manifest for OfficeQA Finance Agent
// See: https://docs.agentbeats.dev/manifest/
{
  manifest_version: "0.1.0",

  // Agent metadata
  name: "OfficeQA Finance Agent",
  description: "RAG agent over US Treasury bulletins (1939-2025). Answers fiscal/financial questions with exact arithmetic grounded in local corpus.",

  // Configuration schema — env vars required at runtime
  config_schema: {
    properties: {
      GOOGLE_CLOUD_PROJECT: {
        type: "string",
        description: "GCP project ID with Vertex AI API enabled",
      },
      MODEL_ID: {
        type: "string",
        description: "LLM model identifier (e.g., gemini-2.0-flash, claude-sonnet-4-6)",
      },
      GOOGLE_APPLICATION_CREDENTIALS: {
        type: "string",
        description: "Path to GCP service account JSON key file",
      },
    },
    required: ["GOOGLE_CLOUD_PROJECT"],
  },

  // Docker program definition
  program: {
    image: "ghcr.io/YOUR_ORG/officeqa-agent:latest",  // TODO: update with actual registry
    entrypoint: ["python", "-m", "src.server"],
    env: {
      GOOGLE_CLOUD_PROJECT: "$config.GOOGLE_CLOUD_PROJECT",
      MODEL_ID: "$config.MODEL_ID",
      GOOGLE_APPLICATION_CREDENTIALS: "$config.GOOGLE_APPLICATION_CREDENTIALS",
    },
    network: {
      endpoints: [{ port: 9009 }],
    },
  },

  // A2A endpoint declaration
  provides: {
    a2a_endpoint: { port: 9009 },
  },
  exports: {
    a2a_endpoint: "a2a_endpoint",
  },
}
```

**Note:** The `image` field has a placeholder `YOUR_ORG` — this will be updated when the actual container registry is configured. This is expected and documented.
