Create `workspace/src/agent.py`. Start with the SYSTEM_PROMPT constant. This prompt is the enforcement mechanism for all agent behavior rules. It must include:

**Section 1: Mandatory Planning Gate (AGT-04, write-todos protocol)**
```
Before calling ANY retrieval tool (route_files, search_in_file), you MUST first call write_todos with:
1. A restatement of the question as you understand it
2. Your planned tool call sequence in order

You may update the todo list mid-run as new evidence changes the plan.
Mark items as completed (status: "completed") as you finish each step.
```

**Section 2: Scratch File Writing Instructions (SCR-02, SCR-03)**
The UID is injected via the user message preamble (not baked into system prompt). Use `{uid}` placeholder notation in instructions:
```
After EACH search_in_file result, APPEND to {uid}/evidence.txt:
  Format: "Source: {file_path}\n{span_text}\nNote: {why this span was selected}\n---\n"

After EACH extract_table_block result, APPEND to {uid}/tables.txt:
  Format: the raw table block as returned by the tool, followed by "\n---\n"

After extracting numeric values from evidence/tables, WRITE to {uid}/extracted_values.txt:
  Format: one line per value, "variable_name = value (unit)" e.g. "defense_1940 = 2602 (millions)"
  EVERY numeric value MUST include its unit. If unit is unclear, write "(unit unknown)".

After EACH calculate/pct_change/sum_values result, APPEND to {uid}/calc.txt:
  Format: expression, labeled inputs with source file, result

After completing all calculation, WRITE to {uid}/verification.txt:
  Content: "verification: pending (Phase 4)"

After calling normalize_answer, WRITE to {uid}/answer.txt:
  Line 1: the normalized answer string
  Line 2: a one-sentence rationale (e.g. "pct_change from 2602 to 3100 over FY1940")
```

**Section 3: Tool Usage Rules (AGT-04)**
```
- NEVER compute percent change with inline arithmetic. ALWAYS use the pct_change tool.
- NEVER generate arithmetic formulas inline. ALWAYS use calculate() for all arithmetic.
- When route_files or search_in_file returns RETRIEVAL_EXHAUSTED, STOP calling that tool.
  Attempt to answer from evidence gathered so far.
  If evidence is insufficient, respond with EXACTLY:
  "I cannot determine the answer from the available corpus."
```

The system prompt must NOT reference `{uid}` literally — the UID is injected via the user message preamble (see run_question). Use relative path notation: the system prompt says "write to {uid}/evidence.txt" and the user message preamble specifies the actual UID value.

IMPORTANT: Per locked decision, the system prompt tells the agent to write these specific file contents:
- evidence.txt: annotated spans (raw text + source path + one-line note on selection reason)
- tables.txt: raw extracted table block only
- extracted_values.txt: name = value (unit) format
- calc.txt: Claude's discretion format — use expression + labeled inputs + result
- answer.txt: normalized answer line 1, one-sentence rationale line 2
- verification.txt: stub "verification: pending (Phase 4)"


   - Calls `get_model(config)` to get the LLM instance
   - Passes all 7 @tool-decorated tools as a list
   - Passes `SYSTEM_PROMPT`
   - Passes middleware list with TWO `ToolCallLimitMiddleware` instances:
     - `ToolCallLimitMiddleware(tool_name="route_files", run_limit=RETRIEVAL_LIMIT, exit_behavior="continue")`
     - `ToolCallLimitMiddleware(tool_name="search_in_file", run_limit=RETRIEVAL_LIMIT, exit_behavior="continue")`
   - Per context decision: limit applies to retrieval tools ONLY; calculate and extract_table_block are UNCAPPED

   - Passes `backend=FilesystemBackend(root_dir="./scratch", virtual_mode=False)` — explicit virtual_mode per Pitfall 3
   - Passes `checkpointer=MemorySaver()` — fresh per agent instance
   - Returns the compiled agent

3. **`run_question(uid: str, question: str, config=None) -> str`:**
   - Calls `prepare_scratch(uid)` from `src.scratch` — wipes and recreates scratch dir (SCR-01, AGT-02)
   - Creates a fresh agent via `create_agent(config)` — this gives a fresh MemorySaver per question (Pitfall 1: MemorySaver does not reset on scratch dir wipe)
   - Prepends UID preamble to user message (per research recommendation Open Question 2):
     ```
     f"Question UID: {uid}\nScratch directory: {uid}/\n\nQuestion: {question}"
     ```
   - Invokes agent:
     ```python
     result = agent.invoke(
         {"messages": [{"role": "user", "content": user_message}]},
         config={"configurable": {"thread_id": uid}},
     )
     ```
   - Extracts final answer from `result["messages"][-1].content`
   - Returns the answer string

4. **Module-level guard:** Do NOT create a global agent instance. Each `run_question` call creates its own agent (fresh MemorySaver). This ensures AGT-02 idempotency.
