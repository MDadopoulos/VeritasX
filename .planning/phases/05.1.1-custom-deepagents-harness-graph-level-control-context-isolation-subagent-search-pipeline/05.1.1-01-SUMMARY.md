## Decisions Made

- `ORCHESTRATOR_SYSTEM_PROMPT` is the source of truth in `harness.py`; `agent.py`'s `SYSTEM_PROMPT` is kept in sync manually to avoid the circular import that would occur if `agent.py` imported from `harness.py` and `harness.py` imported from `agent.py`
- `search-agent` chosen as subagent name (vs `retrieval-agent`) — aligns with plan description, shorter in logs
- `AnthropicPromptCachingMiddleware(unsupported_model_behavior="ignore")` included unconditionally on all middleware stacks — safe no-op for Gemini models, prompt caching benefit for Claude models
- `PatchToolCallsMiddleware` included on all middleware stacks — handles dangling tool calls after interruptions for both models
- Smoke test verification discovered system Python 3.12 (not venv) is invoked by bare `python` command; resolved by using `.venv/Scripts/python.exe` explicitly
