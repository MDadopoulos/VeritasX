<task type="auto">
  <name>Task 2: Integration tests — schema validation, idempotency, state isolation, errors, timeout, health</name>
  <files>workspace/tests/test_server.py</files>
  <action>
Write the following test functions in workspace/tests/test_server.py. All tests use `@pytest.mark.anyio` decorator (NOT `@pytest.mark.asyncio` — we're using pytest-anyio, not pytest-asyncio).

**Test 1: test_run_10_questions_schema (TST-05)**
- POST each of the 10 SAMPLE_QUESTIONS to /run.
- Assert each returns HTTP 200.
- Parse response JSON. Assert it has exactly two keys: "uid" and "answer".
- Assert response["uid"] matches the request uid.
- Assert response["answer"] is a non-empty string.
- Use the mock_run_question fixture so no real agent calls.

**Test 2: test_idempotency_same_uid (HTTP-03)**
- POST a question with uid="IDEM_TEST_001".
- Record the answer from the first response.
- POST the exact same uid and question again.
- Assert second response returns HTTP 200.
- Assert second response answer == first response answer.
- Verify that the mock was called only ONCE (the second request should hit the cache). Use `unittest.mock.call_count` or similar assertion on the mock.

**Test 3: test_idempotency_force_bypass**
- POST a question with uid="FORCE_TEST_001".
- Record answer.
- POST same uid again with `?force=true` query param.
- Assert the mock was called TWICE (force bypasses cache).

**Test 4: test_state_isolation (TST-06)**
- Run two questions with disjoint UIDs: uid_a="ISO_A", uid_b="ISO_B".
- POST uid_a first, then POST uid_b.
- After both complete, read scratch/ISO_B/ files.
- Assert that NO file in scratch/ISO_B/ contains the string "ISO_A" (no cross-contamination).
- Also assert that NO file in scratch/ISO_B/ contains uid_a's answer string.
- The mock_run_question fixture writes deterministic content per UID, so this is checkable.

**Test 5: test_invalid_request_422**
- POST to /run with body `{"question": "no uid here"}` (missing uid).
- Assert HTTP 422.
- Assert response JSON has "reason" key (string).
- Assert response JSON does NOT have "detail" key (FastAPI default — our custom handler overrides it).

**Test 6: test_invalid_request_422_with_uid**
- POST to /run with body `{"uid": "BAD001", "question": 12345}` (wrong type for question).
- Assert HTTP 422.
- Assert response JSON has "uid" == "BAD001" (uid was parseable).
- Assert response JSON has "reason" key.

**Test 7: test_health_endpoint**
- GET /health.
- Assert HTTP 200.
- Assert response JSON has keys: "status", "corpus_files", "model_id", "credentials".
- Assert response["status"] == "ok".
- Assert response["corpus_files"] is an integer >= 0.

**Test 8: test_empty_body_422**
- POST to /run with empty body or invalid JSON.
- Assert HTTP 422.
- Assert "reason" in response JSON.

**Test 9: test_agent_timeout_504**
- Patch `src.server.run_question` with a mock that calls `time.sleep(5)` (or any duration longer than the timeout).
- Set `_app_state["agent_timeout"]` to a very small value (e.g., 0.1 seconds) for this test, OR patch the AGENT_TIMEOUT_SECONDS value before calling the endpoint.
- POST a valid request to /run.
- Assert HTTP 504.
- Assert response JSON has "uid" matching the request uid.
- Assert response JSON has "reason" containing "timed out" (or similar).
- Restore the original timeout value after the test.

**pytest.ini / pyproject.toml note:** If pytest-anyio needs configuration, add `[tool.pytest.ini_options] anyio_backend = "asyncio"` or rely on the fixture. Check if existing pytest.ini has conflicting settings.

**Important:** All tests must handle the lifespan context. The httpx.AsyncClient with ASGITransport should trigger lifespan. If the lifespan's fail-fast checks (GOOGLE_CLOUD_PROJECT, corpus dir) prevent test startup, the mock_run_question or conftest fixtures must set these env vars to valid test values (e.g., set GOOGLE_CLOUD_PROJECT="test-project", CORPUS_DIR to a temp directory with a few dummy .txt files, and patch get_model to return a mock). Add a `mock_startup_deps` fixture that handles this.
