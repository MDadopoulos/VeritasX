<task type="auto">
  <name>Task 2: FastAPI server with POST /run, GET /health, concurrency, idempotency, timeout, error handling</name>
  <files>workspace/src/server.py</files>
  <action>
Create workspace/src/server.py implementing the full HTTP server. Reference 05-RESEARCH.md patterns closely.

**Lifespan (Pattern 1 from research):**
- Use `@asynccontextmanager async def lifespan(app)` — NOT deprecated `@app.on_event`.
- Create `asyncio.Semaphore(MAX_CONCURRENT_RUNS)` inside lifespan (asyncio primitives must be created in running event loop).
- Create `uid_locks: dict[str, asyncio.Lock]` and `uid_locks_guard: asyncio.Lock` inside lifespan.
- **Create the LLM model once at startup:** Call `get_model()` during lifespan and store the returned model object in `_app_state["model"]`. This satisfies HTTP-01 — the expensive model/credentials setup happens once, not per-request. The agent graph is still created per-request (via `create_agent()` in `run_question`) because MemorySaver requires per-invocation isolation, but the underlying LLM object is shared. Also store `_app_state["model_id"]` from the model's metadata and `_app_state["credentials"]` = "ok" if get_model() succeeded.
- Store these plus `corpus_file_count` in a module-level `_app_state: dict` that endpoints read.
- **Fail fast on startup:** Check `GOOGLE_CLOUD_PROJECT` env var exists and is non-empty. Check `CORPUS_DIR` env var (or default `corpus/transformed`) directory exists and contains at least one .txt file. If get_model() raises, let the exception propagate (fail fast). If either env check fails, raise RuntimeError with a clear message — do NOT start accepting requests.
- Read `SERVER_PORT` (default 8000), `SERVER_HOST` (default "0.0.0.0"), `MAX_CONCURRENT_RUNS` (default 4), `AGENT_TIMEOUT_SECONDS` (default 300, i.e. 5 minutes) from env.

**POST /run endpoint:**
- `async def run_endpoint(request: RunRequest, force: bool = False):`
- Step 1: Idempotency check (Pattern 4) — read `SCRATCH_ROOT / uid / "answer.txt"`. If file exists and is non-empty and `force` is not True, return `RunResponse(uid=uid, answer=content.splitlines()[0].strip())`. Log at INFO: `"Returning cached answer for uid={uid}"` (per user decision).
- Step 2: Acquire per-UID lock (Pattern 2). Use `uid_locks_guard` to safely get-or-create an `asyncio.Lock` for this UID in `uid_locks` dict. Then `async with uid_lock:` — if another request for same UID is already running, this waits, then re-checks cache (double-check pattern).
- Step 3: Inside the lock, re-check cache (another request may have just finished). If cached, return.
- Step 4: Acquire global semaphore slot: `async with _app_state["semaphore"]:`.
- Step 5: Run agent **with timeout**: `answer = await asyncio.wait_for(asyncio.to_thread(run_question, uid, request.question), timeout=_app_state["agent_timeout"])` (Pattern 3 — NEVER call run_question directly in async context). The `asyncio.wait_for` wraps the `asyncio.to_thread` call to enforce the timeout.
- Step 6: Return `RunResponse(uid=uid, answer=answer)`.
- **Error handling — three distinct except clauses:**
  - `except asyncio.TimeoutError:` — return `JSONResponse(status_code=504, content={"uid": uid, "reason": "agent timed out"})`. Log at ERROR level.
  - `except Exception as e:` — return `JSONResponse(status_code=500, content={"uid": uid, "reason": f"internal error: {str(e)}"})`. Log at ERROR level with traceback.

**GET /health endpoint:**
- Return `HealthResponse(status="ok", corpus_files=_app_state["corpus_file_count"], model_id=_app_state["model_id"], credentials=_app_state["credentials"])`.
- The credentials and model_id values were already set during lifespan startup.

**Custom 422 handler (Pattern 5):**
- `@app.exception_handler(RequestValidationError)` — import from `fastapi.exceptions`.
- Try to extract `uid` from the raw request body (await request.body(), json.loads, .get("uid")).
- Return JSONResponse status 422, body: `{"uid": uid_or_None, "reason": "...description of validation error..."}`. Omit `uid` key entirely from body if uid was not parseable (per user decision). Use `if uid: body["uid"] = uid` pattern.

**Server invocation (Pattern 7):**
- At bottom of file: `if __name__ == "__main__": uvicorn.run("src.server:app", host=host, port=port)` reading from env vars.
- The app object must be importable as `src.server:app` for direct uvicorn CLI usage too.

**LangSmith tracing (Pattern 6 — NO code change to agent.py):**
- Add a docstring at the top of server.py explaining: "LangSmith tracing is activated by setting LANGSMITH_TRACING=true, LANGSMITH_API_KEY, and LANGSMITH_PROJECT env vars. No code wiring needed — LangChain auto-detects these. The agent already uses thread_id=uid in config, which maps to LangSmith session grouping."
- This is env-var-based activation only. No imports or code changes needed.

**Import structure:**
- `from src.agent import run_question`
- `from src.scratch import SCRATCH_ROOT`
- `from src.schemas import RunRequest, RunResponse, ErrorResponse, HealthResponse`
- `from src.model_adapter import get_model`
- Load dotenv at top: `load_dotenv(Path(__file__).parent.parent / ".env")` matching agent.py pattern.

**Critical pitfalls to avoid (from research):**
- Do NOT create asyncio.Semaphore or asyncio.Lock at module level — creates RuntimeError/DeprecationWarning.
- Do NOT call run_question without asyncio.to_thread — blocks entire event loop.
- Do NOT call prepare_scratch() in the endpoint — run_question already calls it internally.
- Check cache BEFORE acquiring semaphore (avoid holding a slot just to read a file).
- Read only line 1 of answer.txt (not full content) for cache hits.
