# Phase 03.1 Plan 02: Model Adapter + Agent Rewrite Summary

**One-liner:** Rewrote `model_adapter.py` to read MODEL_ID/GCP vars from environment directly and `agent.py` to wire all 7 plain-function tools into `create_deep_agent` with no Config threading.

## What Was Built

Completed the architecture refactor by updating the two orchestration files to match the plain-function tool API delivered in Plan 01.

### model_adapter.py

Replaced `get_model(config: Config | None = None)` with `get_model(model_id: str | None = None)`. The function now reads `MODEL_ID`, `GOOGLE_CLOUD_PROJECT`, and `GOOGLE_CLOUD_LOCATION` directly from `os.environ`. Removed `TYPE_CHECKING` block, `from src.config import Config`, and `from src.config import get_config` lazy import. Default model_id when neither arg nor env var is set: `"gemini-2.0-flash"`.

### agent.py

- **SYSTEM_PROMPT:** Removed the "When route_files or search_in_file returns RETRIEVAL_EXHAUSTED" paragraph and the "I cannot determine" fallback. Kept all scratch file writing instructions (evidence.txt, tables.txt, extracted_values.txt, calc.txt, verification.txt, answer.txt), mandatory planning gate, and tool usage rules.
- **RETRIEVAL_LIMIT constant:** Removed — Deep Agent native max-turns handles this.
- **create_agent():** No config parameter. Calls `get_model()` with no args. Imports all 7 tool functions as plain functions directly. Passes them to `create_deep_agent(tools=[route_files, search_in_file, extract_table_block, calculate, pct_change, sum_values, normalize_answer], ...)`.
- **run_question():** Removed `config` parameter. Signature: `(uid: str, question: str) -> str`.
- **run_question_with_messages():** Removed `config` parameter. Signature: `(uid: str, question: str) -> dict`.
- Removed `TYPE_CHECKING` block, `from src.config import Config`, all `retrieval_wrappers` imports.

**1. [Rule 1 - Bug] Updated test_model_adapter.py to match new env-based get_model API**
- **Found during:** Task 1 verification
- **Issue:** test_model_adapter.py was written for the old `get_model(config: Config)` API. All 10 unit test cases passed Config objects (constructed via `_make_config()`) to `get_model()`. With the new string API, `get_model(config_object)` would use the Config object as the model_id string, failing immediately at `model_id.startswith("gemini-")`.
- **Fix:** Removed `_make_config()` helper and `from src.config import Config` import. Rewrote all test cases to pass `model_id` strings directly and use `patch.dict(os.environ, ...)` to control GOOGLE_CLOUD_PROJECT and GOOGLE_CLOUD_LOCATION. Updated TestDefaultModelId to verify gemini-2.0-flash default (was claude-sonnet-4-6 from old Config default). Added `test_missing_project_env_uses_empty_string` for new env-var default behavior.
