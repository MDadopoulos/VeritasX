# Phase 03.1 Plan 01: Tool Module Refactor — Plain Functions Summary

**One-liner:** Renamed all 5 `_impl` functions to public names and stripped `@tool`/`StructuredTool` wrappers, enabling direct use as `create_deep_agent` callables; deleted `retrieval_wrappers.py`; 368 tests pass.

## What Was Built

Rewrote 5 tool modules to export plain Python functions instead of LangChain StructuredTool instances. This eliminates the Phase 3 workaround pattern (_impl renaming + tool() wrapper + TYPE_CHECKING Config import) that was introduced to work around Pydantic schema introspection issues.

### Changes by file

**calculate.py:** Renamed `_calculate_impl` → `calculate`, `_pct_change_impl` → `pct_change`, `_sum_values_impl` → `sum_values`. Removed `from langchain_core.tools import tool` and 3-line StructuredTool alias block at bottom.

**route_files.py:** Renamed `_route_files_impl` → `route_files`. Deleted `_route_files_agent` thin wrapper. Deleted `route_files = tool(...)` alias. Removed `TYPE_CHECKING` import block and `from src.config import Config`. Added `_CORPUS_DIR = Path(os.environ.get("CORPUS_DIR", ...))` module-level constant using absolute fallback path. Function now takes only `question: str`.

**search_in_file.py:** Renamed `_search_in_file_impl` → `search_in_file`. Removed `from langchain_core.tools import tool` and StructuredTool alias.

**extract_table_block.py:** Renamed `_extract_table_block_impl` → `extract_table_block`. Removed `from langchain_core.tools import tool` and StructuredTool alias.

**normalize_answer.py:** Renamed `_normalize_answer_impl` → `normalize_answer`. Removed `from langchain_core.tools import tool` and StructuredTool alias.

**retrieval_wrappers.py:** Deleted via `git rm`. Counter-wrapper factories (`make_counted_route_files`, `make_counted_search_in_file`) are superseded by Deep Agent native max-turns limit.
