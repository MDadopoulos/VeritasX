1. Create `workspace/src/model_adapter.py`:
   - Import `ChatGoogleGenerativeAI` from `langchain_google_genai`
   - Import `ChatAnthropicVertex` from `langchain_google_vertexai.model_garden`
   - `get_model(config=None)` function:
     - If config is None, call `get_config()` to obtain it
     - Read `model_id` from config
     - If `model_id.startswith("gemini-")`: return `ChatGoogleGenerativeAI(model=model_id, project=config.google_cloud_project, location=config.google_cloud_location)`
     - If `model_id.startswith("claude-")`: return `ChatAnthropicVertex(model_name=model_id, project=config.google_cloud_project, location=config.google_cloud_location)`
     - Otherwise: raise `ValueError(f"Unsupported MODEL_ID: {model_id!r}. Must start with 'gemini-' or 'claude-'")`
   - The function signature accepts an optional config so tests can inject mock config without env vars.

2. Create `workspace/tests/test_model_adapter.py`:
   - Mock `ChatGoogleGenerativeAI` and `ChatAnthropicVertex` constructors using `unittest.mock.patch`
   - Test gemini prefix dispatches to `ChatGoogleGenerativeAI`
   - Test claude prefix dispatches to `ChatAnthropicVertex`
   - Test unset MODEL_ID defaults to claude-sonnet-4-6 (dispatches to ChatAnthropicVertex)
   - Test unsupported prefix raises ValueError with descriptive message
   - Test that project and location are passed through correctly
   - All tests are fully offline — no real Vertex AI calls.

3. Create `workspace/src/corpus_manifest.py`:
   - `load_manifest(csv_paths: list[str]) -> set[str]`: reads each CSV, parses `source_files` column. CRITICAL: source_files uses NEWLINE separators within quoted cells (not comma). Split by whitespace after replacing `\n` with space. Return set of unique filenames.
   - `validate_corpus(config=None) -> tuple[set[str], set[str]]`:
     - If config is None, call `get_config()`
     - Call `load_manifest([config.csv_full_path, config.csv_pro_path])`
     - List actual files in `config.corpus_dir`
     - Compute `missing = required - actual`
     - If `config.corpus_source == "local"` and `missing` is non-empty: log each missing file as WARNING to stderr, then raise `SystemExit(1)` with message "Missing N corpus files in local mode — cannot continue"
     - If missing is empty: log "Corpus manifest OK: N files validated" to stderr
     - Return `(required, missing)`

4. Create `workspace/tests/test_corpus_manifest.py`:
   - Test `load_manifest` correctly parses newline-separated source_files from real CSV files (use the actual `officeqa_full.csv` and `officeqa_pro.csv` — they are in the repo)
   - Test manifest returns exactly 285 unique files
   - Test `validate_corpus` with real corpus dir reports zero missing files
   - Test `validate_corpus` with a temp dir missing one file raises SystemExit in local mode
   - Test that a single-file source_files cell parses correctly
   - Test that a multi-file newline-separated cell parses correctly
   - Use `tmp_path` fixture for the missing-file test, not modifying real corpus
