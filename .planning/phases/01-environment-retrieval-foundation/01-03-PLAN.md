2. Create `workspace/src/tools/search_in_file.py`:

   **`normalize_query(query: str) -> str`:**
   - Expand "FY{digits}" and "FY {digits}" patterns: "FY95" -> "fiscal year 1995", "FY2005" -> "fiscal year 2005". Use same two-digit expansion as route_files (>=30 = 19xx, <30 = 20xx).
   - Strip commas from numbers: "2,602" -> "2602"
   - Normalize dash variants (en-dash, em-dash) to ASCII hyphen
   - Lowercase the result
   - Return the normalized string

   **`build_spans(lines: list[str], window: int = 20) -> list[dict]`:**
   - Build non-overlapping spans from the file lines.
   - Start at line 0. For each span, tentative end = start + window.
   - TABLE BOUNDARY RULE (per user decision — never truncate mid-table):
     - If `lines[end]` starts with `|`, keep extending `end` until you reach a line that does NOT start with `|` (include the full table).
     - Additionally, if the span START is in the middle of a table (lines before start also begin with `|`), extend the span START backward to include the full table from its first `|` line.
     - When a table exceeds 20 lines, the entire table becomes one span — this is correct per user decision.
   - Each span dict: `{"text": "\n".join(lines[start:end]), "start_line": start + 1, "end_line": end, "line_count": end - start}`
   - CRITICAL: advance `i = end` (not `i += window`) to avoid duplicating table rows across spans.
   - Deduplication: if extending a span backward causes overlap with the previous span, merge them into one span (or just start from where the previous span ended and extend forward to capture the full table).

   **`search_in_file(file_path: str, query: str, top_k: int = 5) -> dict | list[dict]`:**
   - Read the file at `file_path` into lines.
   - Build spans using `build_spans(lines)`.
   - Normalize the query using `normalize_query(query)`.
   - Tokenize: split each span text and the normalized query on whitespace (lowercased).
   - Create `BM25Okapi` index from tokenized spans.
   - Get scores for the query tokens.
   - Select top_k spans with score > 0, sorted by score descending.
   - Build result list: each result is a dict with keys:
     - `"text"`: the span text
     - `"source_file"`: the absolute path to the file (as string)
     - `"start_line"`: 1-indexed start line number
     - `"end_line"`: 1-indexed end line number
     - `"bm25_score"`: float score
     - `"regex_fallback"`: False

   **REGEX FALLBACK** (per user decision — structured no-results, not empty list):
   - If BM25 returns zero results AND the original query contains a digit or "FY" (case-insensitive):
     - Build a regex pattern from the normalized query (escape special chars, replace spaces with `\s+`)
     - Search each span's text for a match
     - Return matching spans (up to top_k) with `"bm25_score": 0.0, "regex_fallback": True`

   **NO RESULTS** (per user decision):
   - If BOTH BM25 and regex fallback find nothing:
     - Return `{"error": "no_results", "query": query, "file": file_path, "spans_searched": len(spans)}`
   - Do NOT return an empty list.
